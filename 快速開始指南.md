# 🚀 Podcastfy 快速開始指南

## 📋 完整三步驟工作流程

本指南將引導你完成 Podcastfy 的完整播客生成流程，從內容輸入到最終音頻輸出。

---

## 🔧 環境準備

### 1. 檢查依賴
```bash
# 確保你在正確的目錄
cd /path/to/podcastfy_test

# 激活虛擬環境（如果使用）
source venv/bin/activate  # macOS/Linux
# 或
venv\Scripts\activate     # Windows

# 安裝依賴
pip install -r requirements.txt
```

### 2. 設置 API Key
```bash
# 創建 .env 文件並添加你的 Gemini API Key
echo "GEMINI_API_KEY=你的_GEMINI_API_KEY" > .env
```

### 3. 測試 API 連線
```bash
python test_api.py
```

---

## ⚙️ 配置你的播客

編輯 `podcast_config.yaml`:

```yaml
# 基本設置
basic:
  english_level: "A1"           # A1(探索者) → C2(大師)
  podcast_length: "medium"      # short/medium/long/extra-long

# 輸入來源（支援多種格式）
input:
  sources:
    - "sample_article.txt"      # 本地文件
    - "https://example.com"     # 網頁文章
    - "youtube_url"             # YouTube 影片
  type: "multi"                 # 多資料來源

# 標籤嵌入功能（可選）
tag_embedding:
  enabled: true                 # 開啟 LLM 智能標籤嵌入
  models:
    analysis_model: "gemini-2.5-flash"
    tagging_model: "gemini-2.5-pro"
```

## 🎯 使用方式

### 方式一：自動化生產模式（推薦新手）

一鍵完成所有三個步驟：

```bash
python podcast_workflow.py --mode prod
```

### 方式二：開發模式（推薦進階用戶）

逐步執行，每步確認：

```bash
python podcast_workflow.py --mode dev
```

在每個步驟後，系統會詢問：
- **Step 1 完成**：是否繼續標籤嵌入？（可選 y/n/view 查看腳本）
- **Step 2 完成**：是否繼續音頻生成？

### 方式三：自訂模式（推薦專家用戶）

選擇性執行特定步驟：

```bash
# 僅生成腳本
python podcast_workflow.py --mode custom --steps script

# 僅標籤嵌入（需要指定腳本目錄）
python podcast_workflow.py --mode custom --steps tags \
  --script-dir ./output/scripts/script_20250821_143057

# 僅生成音頻
python podcast_workflow.py --mode custom --steps audio \
  --script-dir ./output/tagged_scripts/tagged_20250821_143145
```

## 📊 步驟詳解

### Step 1: 腳本生成 📝

**功能**：使用 Podcastfy + Gemini API 生成自然對話腳本

**輸入**：文章、URL、PDF、YouTube 影片
**輸出**：`./output/scripts/script_TIMESTAMP/`
- `podcast_script.txt` - 對話腳本
- `metadata.json` - 生成統計

**範例輸出**：
```
<Person1> "Welcome to Natural Conversations - 探索者"
<Person2> "It's wonderful to be back, exploring these fascinating shifts"
```

**手動執行**：
```bash
python generate_script.py
```

### Step 2: LLM 智能標籤嵌入 🏷️

**功能**：使用 LLM 分析對話情境，智能嵌入 SSML 和情感標籤

**輸入**：Step 1 生成的腳本
**輸出**：`./output/tagged_scripts/tagged_TIMESTAMP/`
- `podcast_script_tagged.txt` - 帶標籤腳本
- `original_script.txt` - 原始腳本備份  
- `tag_metadata.json` - 標籤統計

**範例輸出**：
```
<Person1> "[gentle] [conversational] Welcome to Natural Conversations - 探索者 [PAUSE=2s]"
<Person2> "[happy] [warm] It's wonderful to be back, [PAUSE=1s] exploring these fascinating shifts [thoughtful]"
```

**手動執行**：
```bash
python embed_tags.py ./output/scripts/script_TIMESTAMP
```

### Step 3: 音頻生成 🎵

**功能**：使用 Gemini Multi-Speaker TTS 生成最終音頻

**輸入**：Step 2 的帶標籤腳本（或 Step 1 的原始腳本）
**輸出**：`./output/audio/audio_TIMESTAMP/`
- `podcast.wav` - 最終音頻文件
- `script.txt` - 使用的腳本
- `metadata.json` - 生成資訊

**手動執行**：
```bash
python generate_audio.py ./output/tagged_scripts/tagged_TIMESTAMP
```

## 🎨 等級特色

### A1 探索者 🌱
- **風格**：溫和、耐心、支持性
- **語速**：緩慢，充分停頓
- **標籤**：`[gentle]`, `[curious]`, `[PAUSE=2s]`
- **適合**：初學者內容，基礎概念介紹

### B1 討論者 💬  
- **風格**：平衡、參與、分析性
- **語速**：中等，自然節奏
- **標籤**：`[thoughtful]`, `[engaged]`, `[analytical]`
- **適合**：一般討論，中等複雜度話題

### C2 大師 🎓
- **風格**：專業、權威、精巧
- **語速**：快節奏，高效率
- **標籤**：`[sophisticated]`, `[authoritative]`, `[insightful]`
- **適合**：專業內容，複雜概念分析

## 🔍 故障排除

### 常見問題

#### 1. API Key 錯誤
```bash
# 檢查 .env 文件
cat .env

# 測試 API 連線
python test_api.py
```

#### 2. 腳本生成失敗
- 檢查輸入來源是否可訪問
- 確認配置文件格式正確
- 查看 API 配額限制

#### 3. 標籤嵌入跳過
```yaml
# 確保在 podcast_config.yaml 中啟用
tag_embedding:
  enabled: true  # 設為 true
```

#### 4. 音頻生成失敗
- 檢查腳本文件是否存在
- 確認 Gemini TTS API 可用性
- 查看音頻長度限制

### 檢查輸出

#### 查看生成的腳本
```bash
# 查看最新腳本
ls -la ./output/scripts/
cat ./output/scripts/script_LATEST/podcast_script.txt
```

#### 查看標籤統計
```bash
# 查看標籤嵌入統計
cat ./output/tagged_scripts/tagged_LATEST/tag_metadata.json | jq '.tag_embedding.statistics'
```

#### 播放生成的音頻
```bash
# macOS
open ./output/audio/audio_LATEST/podcast.wav

# Linux
xdg-open ./output/audio/audio_LATEST/podcast.wav
```

## 💡 最佳實踐

### 1. 內容準備
- **清晰來源**：確保 URL 可訪問，文件格式正確
- **合適長度**：避免過長內容（建議 < 10,000 字）
- **主題聚焦**：單一主題效果更好

### 2. 配置優化
- **等級選擇**：根據目標聽眾選擇合適等級
- **長度設定**：first-time 建議選擇 `medium`
- **標籤功能**：建議先體驗關閉標籤的效果，再開啟對比

### 3. 成本控制
- **開發模式**：使用 `--mode dev` 逐步確認，避免浪費
- **標籤選擇**：高品質需求才使用 `gemini-2.5-pro` 標籤模型
- **批次處理**：一次處理多個相關內容效率更高

### 4. 品質優化
- **多次生成**：不滿意可調整配置重新生成
- **A/B 測試**：嘗試不同等級和長度設定
- **標籤對比**：對比開啟/關閉標籤的音頻效果
